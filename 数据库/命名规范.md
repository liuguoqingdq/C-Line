## 1) 硬规则（MySQL 层面必须遵守）

1. **只用小写字母 + 数字 + 下划线**：`snake_case`，例如 `user_id`、`created_at`。
    
2. **不要以数字开头**：例如 `1st_name` 不要这样写。
    
3. **避免使用 MySQL 保留关键字**：如 `order`、`group`、`rank` 等；如果不得不用，必须用反引号包裹（不推荐长期这么做）。
    
4. **标识符长度控制**：表名/字段名/索引名不要太长（MySQL 标识符上限通常是 64 字符），团队里最好再设一个更紧的软上限（例如 30–40）。
    
5. **大小写一致**：MySQL 在很多场景下对标识符不区分大小写，但不同 OS/配置会带来差异；统一全小写最省麻烦。
    

---

## 2) 风格规则（让字段“读起来像业务”）

### 2.1 命名要表达“含义”，而不是“类型”

- 推荐：`amount`、`price`、`created_at`
    
- 不推荐：`amount_double`、`created_time_datetime`
    

### 2.2 避免含糊缩写

- 推荐：`description`、`address`、`quantity`
    
- 不推荐：`desc`（还容易跟 SQL 里的 DESC 冲突）、`addr`、`qty`（除非团队统一）
    

### 2.3 同一类字段统一后缀/前缀

例如你所有时间都用 `_at`，所有外键都用 `_id`，可读性会明显提升。

## 3) 常用字段命名约定（强烈建议统一）

### 3.1 主键

- `id`（单表主键最通用）
    
- 或者更明确：`user_id`、`order_id`（当你希望跨表读 SQL 更清晰）
    

建议二选一并坚持，不要混用。

### 3.2 外键

- 一律：`<引用对象>_id`  
    例如：`user_id`、`dept_id`、`parent_id`
    
- 多级关系：`created_by`、`updated_by` 也常见（指向用户 id）
    

### 3.3 时间字段

- 创建时间：`created_at`
    
- 更新时间：`updated_at`
    
- 删除时间（软删）：`deleted_at`
    
- 业务发生时间：`paid_at`、`shipped_at`、`expired_at`
    
- 只有日期：`start_date`、`end_date`
    
- 只有时间：`start_time`、`end_time`
    

> 规则：`*_at` 表示时间点（datetime/timestamp），`*_date` 表示日期（date）。

### 3.4 布尔字段

- 用前缀：`is_` / `has_` / `can_`  
    例如：`is_active`、`has_paid`、`can_refund`
    

> MySQL 里布尔通常落成 `TINYINT(1)`，但字段名保持布尔语义最重要。

### 3.5 状态/枚举字段

- 推荐：`status` / `state` + 明确注释/字典表  
    例如：`status`（0=init,1=active,2=disabled）
    
- 需要多种状态时：`order_status`、`payment_status`
    

### 3.6 数量/金额/比例

- 数量：`*_count` 或 `*_cnt`（团队统一其一）
    
- 金额：`*_amount`（如 `total_amount`）
    
- 比例：`*_rate`（如 `success_rate`）
    
- 百分比：`*_pct`（如 `discount_pct`）
    

---

## 4) 不推荐的字段名模式（常见踩坑）

- `data`、`value`、`info`、`temp`（信息量太低）
    
- `type`（容易变成“万能垃圾桶”；如果要用，至少写成 `user_type`、`event_type`）
    
- `desc`（语义不清且可能与关键字/习惯冲突）
    
- 混用 `camelCase` 与 `snake_case`
    

---

## 5)（加分）索引/约束命名规范（建议也统一）

虽然你问的是字段，但实际团队往往一起规范：

- 主键：`pk_<表名>`
    
- 唯一索引：`uk_<表名>_<字段1>[_字段2]`
    
- 普通索引：`idx_<表名>_<字段1>[_字段2]`
    
- 外键：`fk_<表名>_<字段>__<被引用表名>`
    

例如：

- `idx_orders_user_id_created_at`
    
- `uk_users_email`
    
- `fk_orders_user_id__users`
    

---

## 6) 一份“最小字段模板”（你建表时直接套）

- `id`
    
- `created_at`
    
- `updated_at`
    
- `deleted_at`（可选）
    
- `status`（可选）
    
- 业务外键：`xxx_id`
