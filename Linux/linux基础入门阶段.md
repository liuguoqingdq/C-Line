# 一、基础入门阶段详细讲解

## 1. Linux 基本概念与发行版选择

### 1.1 什么是 Linux？

严格来说：
- **Linux** 本身指的是“内核”（kernel），负责进程调度、内存管理、设备驱动、文件系统等。
- 日常说的 Linux 系统，其实是 **Linux 内核 + GNU 工具链 + 各种软件 + 包管理器 + 默认配置**，通常称为 **“GNU/Linux 发行版（distribution）”**。

你可以简单理解为：
> Windows 有“家庭版、专业版”，Linux 则有“Ubuntu、Debian、Arch、CentOS”等不同的“发行版”。

### 1.2 为什么要学 Linux？

几个现实原因：
- **服务器和云环境的绝对主流**：绝大多数后端服务、数据库、中间件、容器都跑在 Linux 上。
- **开发友好**：自带强大的命令行工具链，编译器、调试器、脚本语言都很方便。
- **可控性和可视性强**：你能看到“系统到底在干什么”，更适合做性能调试、安全、网络等工作。
- **开源和免费**：可以自由安装在各种机器上，适合做实验、搭环境。

### 1.3 常见发行版家族

按“血统”简单分几类：

1. **Debian 系**
   - 代表：Debian、Ubuntu、Linux Mint 等。
   - 包管理：`apt` / `apt-get`。
   - 特点：稳定、包多，新手友好、文档多。

2. **RedHat 系**
   - 代表：RHEL（商业）、CentOS（传统版已停止）、Rocky Linux、AlmaLinux 等。
   - 包管理：`yum` / `dnf`。
   - 特点：企业环境、服务器常见，偏稳定、升级节奏保守。

3. **Arch 系**
   - 代表：Arch Linux、Manjaro。
   - 包管理：`pacman`。
   - 特点：滚动更新、版本新、上手门槛稍高，更适合折腾党。

4. 其他：openSUSE、Gentoo 等，相对小众或更硬核。

### 1.4 初学者推荐

如果你现在只是要：
- 在虚拟机 / 双系统 / WSL 里用 Linux；
- 做开发、练命令行、学网络/系统编程；

**推荐：**
- 桌面：**Ubuntu LTS（长期支持版）** 或 **Linux Mint**。
- 服务器或命令行为主：**Ubuntu Server** 或 **Debian stable**。

有了一个“主力发行版”之后，其他发行版的差别，主要在包管理命令和一些配置路径上，本质上 Linux 的核心概念是通用的。

## 2. 图形界面 vs 命令行

### 2.1 图形界面（GUI）的组成

在 Linux 上，图形界面一般由几层组成：
- 显示协议：**X11** 或 **Wayland**；
- 桌面环境（DE）：例如 **GNOME、KDE Plasma、XFCE** 等；
- 应用程序：文件管理器、浏览器、终端模拟器（terminal）等等。

桌面版发行版（如 Ubuntu Desktop）通常已经帮你配好了这些东西。

### 2.2 命令行（CLI）是什么？

命令行环境一般包含：
- **TTY/终端设备**：系统提供的“字符界面”；
- **终端模拟器**：图形界面中的 `gnome-terminal`、`konsole` 等，就是一个“窗口版终端”；
- **Shell**：真正解释命令的程序，比如 `bash`、`zsh`，它读取你的输入，解析后调用底层程序。

你平时看到的：
```bash
user@host:~$
```
这行就是 **Shell 的提示符（prompt）**，你在后面敲命令、按回车，Shell 就帮你执行。

### 2.3 为什么 Linux 学习要“命令行优先”？

原因本质只有几点：
- **自动化和脚本化**：命令可以写成脚本重复执行，适合做批量任务；
- **不依赖图形环境**：服务器上通常没有桌面环境，只有 SSH + Shell；
- **更精确可控**：很多配置只有命令行或直接改配置文件才搞得定；
- **通用性强**：不同发行版的桌面体验差异很大，但命令行和工具链高度相似。

建议的学习姿势：
- 本地装带桌面的 Linux（方便上手）；
- 但 **尽量多用终端做事**，比如用 `cd`、`ls` 管理文件，用 `vim` 编辑配置，用 `ssh` 登录远程机器。

## 3. 终端与 Shell 基础（以 bash 为主）

### 3.1 Terminal / Shell / TTY 有啥区别？

简单类比：
- **TTY / 终端设备**：一个“键盘 + 屏幕”的抽象；
- **终端模拟器**：图形界面里的“终端窗口”应用；
- **Shell（bash/zsh）**：跑在终端里的“命令解释器”，它负责：
  - 显示提示符；
  - 读取你的命令；
  - 帮你拆分命令、参数和重定向；
  - 调用真正的可执行程序。

### 3.2 命令的基本形式

通用形式：
```bash
命令名 [选项] [参数]
```
例如：
```bash
ls -l /home
# ls   命令
# -l   选项（长格式）
# /home 参数（要查看的目录）
```

习惯：
- **选项** 通常是 `-a` 这种短选项，或 `--all` 这种长选项；
- 多个短选项可以合并：`ls -la` 等价于 `ls -l -a`。

### 3.3 路径与当前工作目录

几个符号要记住：
- `/`：根目录（文件系统的最顶层）；
- `.`：当前目录；
- `..`：上一级目录；
- `~`：当前用户的 home 目录，例如 `/home/yourname`。

路径有两种写法：
- **绝对路径**：从根 `/` 开始，例如 `/etc/passwd`；
- **相对路径**：相对于当前工作目录，例如当前在 `/home/user`，`cd ..` 就是到 `/home`。

### 3.4 常用快捷键（强烈建议早点熟悉）

在 bash 中：
- `Tab`：自动补全命令或文件名；
- `↑` / `↓`：在历史命令中翻动；
- `Ctrl + A`：光标移到行首；
- `Ctrl + E`：光标移到行尾；
- `Ctrl + U`：删除光标到行首；
- `Ctrl + K`：删除光标到行尾；
- `Ctrl + R`：在历史命令中搜索（非常好用）；
- `Ctrl + C`：中断当前命令；
- `Ctrl + Z`：挂起当前命令（放到后台，稍后可用 `fg` 恢复）。

### 3.5 PATH 与找命令的规则

当你敲下：
```bash
ls
```
Shell 会按 `PATH` 环境变量中包含的一系列目录去依次查找 `ls` 这个可执行文件。

相关命令：
- `echo $PATH`：查看 PAheTH 内容；
- `which ls`：告诉你 `ls` 的实际路径；
- `type ls`：告诉你 `ls` 是“内建命令”还是可执行文件。

理解 PATH 对后面“自己写的小工具放哪”“命令为什么找不到”很重要。

## 4. 基本命令使用习惯与帮助系统（man / --help）

### 4.1 最核心的一组基础命令

先记住这一组（文件与目录操作）：

- `pwd`：打印当前工作目录；
- `ls`：列出目录内容（`ls -l`、`ls -a` 常用）；
- `cd`：切换目录，如 `cd /etc`、`cd ..`、`cd ~`；
- `mkdir`：新建目录，如 `mkdir test`；
- `rmdir`：删除空目录；
- `rm`：删除文件/目录（`rm file`，`rm -r dir`）；
- `cp`：复制文件/目录（`cp a b`，`cp -r dir1 dir2`）；
- `mv`：移动/重命名（`mv a b`）。

查看文件内容：
- `cat file`：直接输出整个文件；
- `less file`：分页查看（可上下翻页，按 `q` 退出）；
- `head -n 20 file`：看前 20 行；
- `tail -n 20 file`：看后 20 行；
- `tail -f file`：实时“追踪”文件末尾（看日志很常用）。

> 实用建议：文件稍微一大就不要用 `cat`，改用 `less`，否则屏幕刷过去什么都看不清。

### 4.2 `rm -rf` 的危险

`rm` 不会把东西送到“回收站”，默认就是直接删：

- `rm file`：删除文件；
- `rm -r dir`：递归删除目录；
- `rm -rf /something`：不询问、递归、强制删除，非常危险。

初学时习惯在重要地方多打一眼路径，或者先用 `ls` 看看目录内容再删。

### 4.3 帮助系统：`man` 与 `--help`

Linux 上**几乎所有命令都有文档**，你只需要知道去哪看：

1. `man`（manual）

   ```bash
   man ls
   ```

   - 上下键翻页；
   - `/关键字` 搜索，例如 `/time`；
   - `n` 跳到下一个匹配；
   - `q` 退出。

   `man` 是最权威、最完整的文档，第一次接触命令时建议至少扫一眼 SYNOPSIS（语法）和 DESCRIPTION（描述）。

2. `--help` 选项

   很多命令支持：

   ```bash
   ls --help
   ```

   会在终端直接打印简要的用法说明和常用选项列表，比 `man` 轻量；适合快速查某个选项的意义。

3. 其他帮助命令

   - `whatis ls`：一行简介；
   - `apropos keyword`：按关键字在所有 man page 标题中搜索，比如 `apropos network`。

### 4.4 一个良好的学习习惯

- 每次遇到新命令，先用：`man 命令` 或 `命令 --help` 看一眼；
- 别死记语法，多通过实际任务去“查文档+尝试”；
- 遇到复杂组合（管道、重定向）时，可以把命令拆开，逐步确认每一段在做什么。

只要你养成“遇到不会的先查 man / --help，再自己试两把”的习惯，后面学 Linux 会越来越顺。
